Revitalized Open Source Software

Placeholder Goal:

To revitalize Abandoned, Legacy Open Source Software

Current working definition of Abandoned

  • Project is not part of an existing community or foundation
  • Patches are not being accepted by the upstream source

Current Working definition of legacy:

  • Upstream is not updated in >= 10 years
  • Software is foundational to other pieces of software
For the definition of Open Source Software, read the OSI definition

FAQ

Why did you start this?

I started looking through software running on my computer and discovered libraries and programs that were based on sources that appeared abandoned, with the maintainers of my operating system heavily patching them for use.

What does a revitalized open source project look like?

A successfully revitalized open source project has the following:

  • An active Issue Tracker
  • Continuous Integration Build system with automated tests
  • recent meaningful code changes
  • A way to confidentially disclose vulnerabilities

I don't want to work on Github, could you use a different platform?

While Github does have a lion's share of the market, some people prefer different platforms. Therefore I have reserved the RevitalizedOSS namespace on:

  • Bitbucket
  • Gitlab
  • Phabricator

I also have the domain revitalizedoss.com, so with donations I could set up additional infrastructure.
